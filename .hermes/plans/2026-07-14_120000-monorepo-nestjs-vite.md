# beton-boi Monorepo Implementation Plan

> **For Hermes:** Use plan mode tasks sequentially. Each task is self-contained with explicit file paths and code.

**Goal:** Convert beton-boi from a Next.js boilerplate into a monorepo with NestJS backend + multiple Vite React clients, with a single build command that produces a self-contained deployable folder.

**Architecture:** Yarn workspaces monorepo. NestJS serves the API (`/api/*`) and, in production, serves each client's static build (`/student/*`, `/teacher/*`, `/admin/*`). A root `yarn build:all` script compiles everything into `build-output/` — a self-contained folder you zip, scp, unzip, and run.

**Tech Stack:** TypeScript 5, NestJS, Vite + React 19, Tailwind CSS v4, TypeORM + PostgreSQL, Vitest, Yarn 1 workspaces

```typescript
import { NestFactory } from "@nestjs/core";
import { NestExpressApplication } from "@nestjs/platform-express";
import { AppModule } from "./app.module";
import * as express from "express";
import { join } from "path";

async function bootstrap() {
  const app = await NestFactory.create<NestExpressApplication>(AppModule);

  // Global prefix for API routes
  app.setGlobalPrefix("api");

  // In production, serve client static builds
  if (process.env.NODE_ENV === "production") {
    const clients = ["student", "teacher", "admin"];

    for (const client of clients) {
      const distPath = join(__dirname, "..", "..", `client-${client}`, "dist");

      // Serve static assets
      app.use(`/${client}`, express.static(distPath));

      // SPA fallback: any unknown route under /client/ serves index.html
      app.use(`/${client}`, (_req, res) => {
        res.sendFile(join(distPath, "index.html"));
      });
    }

    // Root redirect to /student/ (or a landing page)
    app.use("/", (_req, res) => {
      res.redirect("/student/");
    });
  }

  await app.listen(process.env.PORT ?? 3000);
}

bootstrap();
```

**Step 1:** Write `server/src/config/env.validation.ts`

```typescript
import { plainToInstance } from "class-transformer";
import { IsNotEmpty, IsString, validateSync } from "class-validator";

class EnvironmentVariables {
  @IsString()
  @IsNotEmpty()
  DATABASE_URL!: string;

  @IsString()
  @IsNotEmpty()
  JWT_SECRET?: string;
}

export function validate(config: Record<string, unknown>) {
  const validatedConfig = plainToInstance(EnvironmentVariables, config, {
    enableImplicitConversion: true,
  });

  const errors = validateSync(validatedConfig, {
    skipMissingProperties: false,
  });

  if (errors.length > 0) {
    throw new Error(errors.toString());
  }

  return validatedConfig;
}
```

**Step 1:** Write `server/src/common/filters/http-exception.filter.ts`

```typescript
import {
  ExceptionFilter,
  Catch,
  ArgumentsHost,
  HttpException,
  HttpStatus,
  Logger,
} from "@nestjs/common";
import { Request, Response } from "express";

@Catch()
export class AllExceptionsFilter implements ExceptionFilter {
  private readonly logger = new Logger(AllExceptionsFilter.name);

  catch(exception: unknown, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse<Response>();
    const request = ctx.getRequest<Request>();

    const status =
      exception instanceof HttpException
        ? exception.getStatus()
        : HttpStatus.INTERNAL_SERVER_ERROR;

    const message =
      exception instanceof HttpException
        ? exception.message
        : "Internal server error";

    this.logger.error(
      `${request.method} ${request.url} → ${status}: ${message}`,
      exception instanceof Error ? exception.stack : undefined,
    );

    response.status(status).json({
      statusCode: status,
      message,
      timestamp: new Date().toISOString(),
      path: request.url,
    });
  }
}
```

**Step 2:** Write `server/src/common/pipes/validation.pipe.ts`

```typescript
import {
  PipeTransform,
  Injectable,
  ArgumentMetadata,
  BadRequestException,
} from "@nestjs/common";
import { validate } from "class-validator";
import { plainToInstance } from "class-transformer";

@Injectable()
export class ValidationPipe implements PipeTransform {
  async transform(value: unknown, { metatype }: ArgumentMetadata) {
    if (!metatype || !this.toValidate(metatype)) {
      return value;
    }

    const object = plainToInstance(metatype, value);
    const errors = await validate(object);

    if (errors.length > 0) {
      const messages = errors.map((err) =>
        Object.values(err.constraints ?? {}),
      );
      throw new BadRequestException(messages.flat());
    }

    return object;
  }

  private toValidate(metatype: Function): boolean {
    const types: Function[] = [String, Boolean, Number, Array, Object];
    return !types.includes(metatype);
  }
}
```

## Phase 5: Build and Deploy Pipeline

### Task 5.1: Create build-all script

**Objective:** Root script that builds everything and assembles `build-output/`.

**Files:**
- Create: `scripts/build-all.sh`

**Step 1:** Write `scripts/build-all.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

ROOT_DIR="$(cd "$(dirname "$0")/.." && pwd)"
OUTPUT_DIR="$ROOT_DIR/build-output"

echo "==> Cleaning previous build..."
rm -rf "$OUTPUT_DIR"
mkdir -p "$OUTPUT_DIR"

echo "==> Building server..."
yarn workspace server build

echo "==> Building client-student..."
yarn workspace client-student build

# Future clients:
# echo "==> Building client-teacher..."
# yarn workspace client-teacher build

echo "==> Assembling build-output..."

# Server: copy dist + package.json + install prod deps
mkdir -p "$OUTPUT_DIR/server"
cp -r "$ROOT_DIR/server/dist" "$OUTPUT_DIR/server/dist"
cp "$ROOT_DIR/server/package.json" "$OUTPUT_DIR/server/package.json"

echo "==> Installing production dependencies for server..."
(cd "$OUTPUT_DIR/server" && yarn install --production --frozen-lockfile)

# Clients: copy dist folders
for client_dir in "$ROOT_DIR"/client-*/; do
  client_name=$(basename "$client_dir")
  if [ -d "$client_dir/dist" ]; then
    echo "==> Copying $client_name static files..."
    cp -r "$client_dir/dist" "$OUTPUT_DIR/$client_name"
  fi
done

# Copy start script and env example
cp "$ROOT_DIR/scripts/start.sh" "$OUTPUT_DIR/start.sh"
chmod +x "$OUTPUT_DIR/start.sh"
cp "$ROOT_DIR/.env.example" "$OUTPUT_DIR/.env.example"

echo "==> Done! Build output is at $OUTPUT_DIR"
echo "    Deploy: zip -r deploy.zip build-output/"
echo "    Run on server: unzip deploy.zip && cp .env.example .env && ./start.sh"
```

## Risks and Notes

- **Yarn 1 workspaces:** Yarn 1 hoists all dependencies to root `node_modules/`. Works fine for monorepos of this size. Don't switch to Yarn 3+ unless you need PnP.
- **NestJS CommonJS:** NestJS requires CommonJS module output. The server tsconfig overrides the base to use `"module": "CommonJS"`. Clients use ESM (Vite's native format).
- **build-output node_modules:** Installing `--production` in `build-output/server/` only includes runtime deps (no devDependencies). This keeps the zip small.
- **Path handling in production:** The server's `main.ts` uses `join(__dirname, "..", "..", "client-student", "dist")` because `__dirname` is `build-output/server/dist/` at runtime. Going up two levels reaches `build-output/`.
- **.env location:** In production, the .env lives at `build-output/.env`, not `build-output/server/.env`. The ConfigModule path (`../.env`) accounts for this.
- **SPA routing:** Each client has client-side routing (e.g., React Router). The express static + fallback middleware ensures refreshing `/student/some-page` serves `index.html` correctly.
